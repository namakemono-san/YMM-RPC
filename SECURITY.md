# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in YMM-RPC, please report it privately.

**Do not open GitHub Issues for security-related problems.**

Instead, report security issues by email:

security@nmkmn.moe

When reporting a security issue, please include:

- A clear description of the vulnerability
- Steps to reproduce the issue, if possible
- Any relevant impact or context

Email is reserved **only** for security-related reports.

Do not use this email address for bug reports, feature requests, questions,
or general support. All non-security communication must go through GitHub Issues.

## Scope

This policy applies to:

- Security vulnerabilities
- Potential security impacts
- Sensitive information disclosure

Non-security bugs should be reported via GitHub Issues.

## Disclosure

Please allow reasonable time for the issue to be reviewed and addressed
before making any information public.
